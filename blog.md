# 블로그 공지 및 내부 보고: 이미지 송수신 속도 개선 및 GIF 재생 안정성 확보

최근 서비스 장기 운영 시 가끔 웹 페이지 로딩이 멈추거나, GIF가 포함된 게시물에 접속이 불가능했던 현상 및 느린 파일 전송 속도를 해결하기 위해 엔진 패치를 진행했습니다.

---

## 1. 이미지 및 미디어 송수신 속도 개선 (페이로드 최적화)

### 문제점
* 기존 파일 전송 프로토콜(TASFA)은 네트워크 대역폭을 아끼기 위해 데이터를 압축하여 전송하도록 설계되어 있었습니다.
* 그러나 브라우저가 지원하는 압축 포맷(예: Gzip)과 서버 내부에서 시도하는 최신 압축 포맷(Zstd, Brotli) 간의 협상 과정에 논리적 버그가 있었습니다.
* 이로 인해 브라우저가 Gzip만 지원할 때도 서버의 압축 데이터가 최종 단계에서 버려지고, 결국 모든 파일이 **압축되지 않은 원본 상태(Raw)로 전송**되는 문제가 발생하여 속도가 매우 저하되었습니다.

### 해결 방안
* 클라이언트(브라우저)가 보낸 지원 압축 목록을 서버가 직접 인지하도록 최적화했습니다.
* 클라이언트의 지원 여부에 맞춰 서버가 최적의 압축 필터(`Zstd` → `Brotli` → `Gzip` 순)를 우선순위대로 즉각 적용하여 전달하도록 구조를 개선했습니다.
* **효과**: 이제 모든 이미지와 파일 청크들이 브라우저가 압축 해제할 수 있는 형태로 정상 압축되어 날아가므로, 페이로드 크기가 크게 줄어들고 다운로드 속도가 비약적으로 상승했습니다.

## 2. GIF 미디어 멈춤 현상 해결 및 미니 루프 플레이어 도입

### 문제점
* 무거운 GIF 애니메이션을 로딩할 때, 서버 내부에서 썸네일을 생성하는 과정이 너무 오래 걸리고 다른 요청을 모두 막아서(Global Lock) 전체 웹 서비스가 마비되는 현상이 있었습니다.
* 또한 이전 조치로 인해 GIF 프리뷰가 제대로 재생되지 않거나 클릭이 안 되는 불편함이 있었습니다.

### 해결 방안
* **미니 루프 플레이어 전환**: 무겁고 성능을 갉아먹는 GIF 포맷 대신, 가볍게 변환된 MP4 비디오 형식으로 브라우저에 전달합니다.
* **사용자 친화적 조작**: 일반 동영상과 달리 로딩 시 소리 없이 자동 재생(Autoplay, Muted, Loop)되며, 한 번 클릭하면 일시 정지하고, 다시 클릭하면 다시 계속 도는 직관적인 작동 구조를 적용했습니다.
* **대용량 스트리밍 적용**: 단순 전체 로드 방식 대신 비디오 분할 스트리밍 방식인 `data-tasfa-src` 경로를 정확히 타게 만들어 대용량 GIF가 포함된 게시물도 렉 없이 부드럽게 로딩됩니다.
* **원본 다운로드 보존**: 게시글 내부에서 원본 GIF 파일을 그대로 다운로드할 수 있는 링크를 제공하여 편의성을 유지했습니다.

## 3. GIF 루프 비디오의 병렬 다중 연결(Multi-connection) 다운로드 적용 및 포스터 제공

### 문제점
* GIF를 비디오로 변환하여 렌더링한 후, 브라우저 기본 `<video>` 태그의 단순 Range Request에 의존해 스트리밍을 수행하도록 변경했었습니다.
* 그러나 이 방식은 브라우저의 단일 커넥션으로만 동작하여, TASFA 프로토콜 고유의 설계 목적인 **다중 연결(Multi-connection) 병렬 다운로드** 및 **다중 청크(Multi-chunk) 오류 복구(HTP 및 XOR Parity 복구)** 이점을 전혀 살리지 못하고 로딩 속도가 느렸습니다.
* 또한 비디오 데이터가 백그라운드에서 다운로드되어 첫 프레임이 디코딩되기 전까지 플레이어가 검은 화면이나 투명한 빈 공간으로 보여 레이아웃이 튀는 현상이 있었습니다.

### 해결 방안
* **TASFA 병렬 다운로드 엔진 탑재**: 컨트롤러가 없는 GIF 루프 비디오 태그에 대해서는 이미지와 마찬가지로 **TASFA의 병렬 다중 청크 다운로드 엔진(`fetchBlobViaTasfa`)을 타도록 프론트엔드(`tasfa-download.js`)를 개선**했습니다.
* **초고속 병렬 다운로드**: 이제 비디오 데이터도 수십 개의 연결로 동시에 나누어 초고속으로 다운로드된 후 메모리에 Blob URL로 로드되어 재생됩니다.
* **정적 WebP 포스터 썸네일 제공**: 비디오가 백그라운드에서 완전히 준비될 때까지 사용자에게 즉각 노출될 정적 WebP 포스터 이미지(`data-tasfa-poster`)를 비디오 태그에 연동했습니다.
* **효과**: 비디오 스트림 로딩 중에 포스터 썸네일이 화면을 채워 레이아웃이 깨지지 않으며, 로딩 완료 즉시 버벅임 없이 극도로 매끄럽게 자동 재생이 시작됩니다.
