# fly.board

![fly.board logo](img/logo.png)

> Один из немногих простых движков блогов, работающих с **~82 МБ RSS** в простое (при 4 воркерах; при одном воркере сохраняет **90-200 МБ**) и **~117 МБ** при C10k (10 000 одновременных соединений).  
> Лёгкий гибрид форума и блога на основе C-фреймворка CWIST с поддержкой HTTPS/3, Argon2id, PQC-подписей и NATS.
>
> **Fairly small, greater usability.**  
> TASFA намеренно жертвует RPS. Пофрагментное шифрование, валидация HTP, битовые карты сессий и адаптивный пейсинг гарантируют, что загрузка не прервётся даже в деградированной сети, при этом блокируя DoS-атаки и подмену чанков в погоне за максимально надёжной передачей.  
> PQC-подписи принимают на себя оверхед ML-DSA-65, обеспечивая обнаружение подделок тела поста в эпоху квантовых компьютеров.  
> Одновременная поддержка HTTP/1.1, HTTP/2 и HTTP/3 отказывается от пиковой производительности одного протокола ради универсальной доступности из любого firewall, прокси и устаревшего устройства.

## Возможности

- **Эффективное использование памяти** — реализация на C со стеком и кучей. **~82 МБ RSS** в простое; максимальный RSS **~117 МБ** при 10 000 одновременных соединениях (C10k).
- **Современный транспорт** — TLS 1.3 + HTTP/3 (QUIC) по умолчанию. Опциональный ECH (Encrypted Client Hello).
- **Безопасная аутентификация** — клиентский прехеш SHA-512 + серверный **Argon2id** (KDF из OpenSSL 3). JWT-куки для сессий.
- **Гибрид форум/блог** — Markdown-посты на основе slug + несколько досок + вложенные комментарии.
- **Предпросмотр в реальном времени** — мгновенный рендеринг предпросмотра на стороне сервера из Markdown-редактора.
- **PQC-подписи** — прикрепление и проверка постквантовых криптографических подписей к постам.
- **Файловое хранилище** — ≤1 МБ в SQLite, файлы больше — на томе. Автовстраивание изображений/видео/аудио.
- **Интеграция NATS** — распределённый шлюз сообщений через переменную окружения `NATS_URL`.
- **Тёмная тема** — переключение темы на основе Cookie с динамическими CSS-переменными.

## Сборка

```sh
make
./keygen.sh
```

Зависимости:
- [CWIST](https://github.com/religiya-serdtsa/cwist) — TLS 1.3 / HTTP/3 (QUIC) обрабатывается встроенным BoringSSL внутри CWIST; дополнительная установка не требуется.
- OpenSSL 3.x (Argon2id KDF)
- ngtcp2 / nghttp3 (HTTP/3)
- cJSON, SQLite3

`Makefile` клонирует и собирает `third_party/md4c` в виде статической библиотеки.

## Запуск

```sh
./fly_board
```

Порт по умолчанию берётся из значения `port` в файле `blog.settings` (по умолчанию 9443).

```text
https://localhost:9443
```

HTTP/3 слушает на том же порту по UDP.

### Включение ECH (опционально)

```sh
BLOG_ECH_KEY=ech/server.ech ./fly_board
# или
BLOG_ECH_DIR=ech ./fly_board
```

Если сборка OpenSSL не поддерживает ECH, в лог будет записано предупреждение, а сервер продолжит работу с обычным HTTPS/3.

### Интеграция NATS (опционально)

```sh
NATS_URL=nats://localhost:4222 ./fly_board
```

## Ключевые возможности

| Возможность | Путь | Описание |
|---------|------|-------------|
| Главная | `/` | Список последних постов |
| Доски | `/boards` | Управление несколькими досками (только для администраторов) |
| Пост | `/post/:slug` | рендеринг md4c Markdown + комментарии + вложения |
| Вход/Регистрация | `/login`, `/register` | Argon2id + JWT-куки |
| Профиль | `/profile` | Никнейм, биография, аватар, дата регистрации |
| Настройки аккаунта | `/account/settings` | Редактирование профиля |
| Смена пароля | `/account/password` | Проверка текущего пароля, повторный хеш Argon2id |
| Админка | `/admin/users` | Изменение ролей пользователей, удаление пользователей |
| Файловое хранилище | `/files` | Загрузка/скачивание/удаление |

## Конфигурация

- `blog.settings` — заголовок блога, подзаголовок, подвал, порт
- `admin.settings` — учётная запись администратора (2 строки: `username`\n`password`)

## База данных

SQLite3 (`data/blog.db`). Схема автоматически мигрируется при запуске приложения.

```
users       — аккаунты, хеши Argon2id, роли, профили
boards      — название/slug/описание доски/admin_only
posts       — тело Markdown, PQC-подпись, краткое содержание
files       — путь/размер/MIME вложения
comments    — вложенные комментарии (target_type, parent_id)
board_permissions — права доступа к приватным доскам
```

## Архитектура

```
CWIST (HTTP/3, TLS 1.3)
  ├── src/auth/     — Argon2id, JWT, сессии
  ├── src/db/       — SQLite3 CRUD
  ├── src/handlers/ — маршрутизация/бизнес-логика
  ├── src/render/   — cwist_html_element SSR + md4c
  ├── src/crypto/   — PQC подпись/проверка
  └── src/nats/     — Pub/Sub сообщений
```

**Ключевые преимущества бенчмарка C10k**
- **Эффективность памяти**: RSS ниже 660 МБ при 10,000 соединениях (~66 КБ на соединение)
- **Нулевой дисковый I/O**: Major page faults 1, Swaps 0, FS inputs 0 — чистая обработка в памяти под нагрузкой
- **Высокая утилизация CPU**: Стабильное ~200% использования CPU
- **Долгосрочная стабильность**: 21.72 с непрерывной нагрузки C10k и корректный выход (код 0)
- **Безопасность данных**: SQLite безопасно сохраняет данные после SIGINT (8 FS outputs)
